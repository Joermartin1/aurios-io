Chapter 11 — AuriOS: The Complete System

Everything described in Chapters 3 through 10 lives somewhere in this architecture. This chapter maps the complete system — every component, its current build status, and the roadmap to completion.

CONFIDENTIAL · NDA-Gated · Founding Circle Edition · Joe R Martin · 2026

Everything described in Chapters 3 through 10 lives somewhere in this architecture. The Member interface. The orchestration layer. The memory tiers. The Partner AIs. The agents. The governance foundation. This chapter maps the complete system — every component, its current build status, and the roadmap to completion.

The AuriOS™ Complete System diagram below is the master reference. The sections that follow walk through each layer in detail.

The AuriOS™ Complete System. Member → Ask Auri™ → Auri™ Orchestration → Partner AIs + Custos Aurum™ → Agents + Outside World → Memory Entity governance.

11.1 The Master Architecture Diagram

The AuriOS™ Complete System diagram above maps every major component and its current build status. LIVE components are in production. IN PROGRESS components are actively being built. PLANNED components are designed and sequenced. The architecture is complete. The build is underway.

The data flow runs top to bottom: the Member interacts through Ask Auri™, which passes queries to the Auri™ Orchestration Layer, which draws on Memory and Partner AIs to assemble a response, while Agents extend the system's reach into the outside world. The Memory Entity sits at the foundation — not as a technical component but as the governance infrastructure that makes the entire system trustworthy.

11.2 Component Inventory

—Interface Layer — Ask Auri™ is the primary Member interface — text input, voice input through the Auri™ Voice Layer, and the Pre-Submit Intelligence Layer containing Query Preview™ and Prompt Assist™. Status: LIVE for text input. Voice Layer and Pre-Submit Intelligence Layer: PLANNED.
—Orchestration Layer — The Auri™ Orchestration Layer classifies every query, retrieves relevant memory from Custos Aurum™, assembles the context window, routes to Partner AIs, synthesizes responses, and writes every session to Real-Time Session Storage™. Status: LIVE for core routing and synthesis. Real-Time Session Storage™: IN PROGRESS — Priority Build Item number one.
—Memory Layer — Custos Aurum™ — the cooperative Memory Entity Infrastructure — is the primary production instance. 99.999% uptime. Encrypted. Member-owned cooperative infrastructure. The local backup instance maintains a complete synchronized copy on the Member's machine for speed when at home and redundancy always.

Within Custos Aurum™: the L1 Hot Tier of curated promoted memory records is built and always searched. The L2 Warm Tier of semi-curated project summaries is planned. The L3 Cold Tier of the complete raw corpus is stored and preserved intact, not yet fully searchable by meaning. FTS5 keyword search is built. ChromaDB vector search, hybrid search, and re-ranking are planned. The Ingestion Pipeline — parsing, chunking, candidate extraction, Member review, and promotion — is built. The MQI™ and Auri™ Tool Interface are built.
—Partner AI Layer — The current development and testing stack: OpenAI GPT-5, Anthropic Claude, Google Gemini, xAI Grok, Perplexity Sonar. These were selected for accessibility and capability during the build phase — not as a result of structured evaluation. Final Partner AI selection will follow rigorous assessment. IBT Mode™ basic routing is in progress. Full IBT Mode™ adjudication is planned.
—Agent Layer — The Update Agent is live — the backbone of Self-Build. The Health Agent, KTM Telemetry and Intelligence System, CIIM™, and Dorada Agent are planned. PAIPy™ interface, Auri Identity™ layer, and MQI™ Public Profile are planned.
—Developer Tools — The Console — the developer interface — is built. The Fuel Gauge token budget display and Injection Visibility Panel are planned.

11.3 Complete Data Flow — A Query from Start to Finish

A single IBT Mode™ query, from Member keystroke to synthesized response:

The Member types a query into Ask Auri™. As the query forms, Query Preview™ classifies the query type and displays a routing suggestion in real time — before Enter is pressed. Prompt Assist™ simultaneously retrieves relevant memory from Custos Aurum™ and offers a substantively improved version of the query incorporating context the Member has not yet typed. The Member accepts, modifies, or dismisses both suggestions.

The Member presses Enter. The final query — with routing confirmed — is sent to the Auri™ Orchestration Layer. Auri™ assembles the context window: system prompt first, injected memory from Custos Aurum™ (typically 500–2,000 tokens of targeted personal context) positioned for maximum model attention, conversation history from the current session, and the query last.

In Single Mode, the assembled context is sent to one Partner AI. The response is returned, presented to the Member, and written to Custos Aurum™ Real-Time Session Storage™ as it arrives.

In IBT Mode™, Auri™ customizes the prompt for each active Partner AI and queries all of them in parallel. Each model receives the same memory context but a prompt tailored to its specific strengths. Responses return simultaneously. Auri™ adjudicates — identifying points of agreement as high-confidence findings, surfacing disagreements explicitly, identifying unique contributions from each model, and synthesizing a single coherent response with key claims attributed to their source models.

The synthesized response is presented to the Member. It is added to the context window. Follow-up questions build on it without rebuilding context. Every turn — query and response — is written to Custos Aurum™ Real-Time Session Storage™ as it happens. The session is permanent. The conversation builds on all previous conversations. The chatbot problem is solved.

—IN PROGRESS: Real-Time Session Storage™ — Priority Build Item number one. IBT Mode™ basic adjudication.

—PLANNED — Phase 2: Full IBT Mode™ adjudication, Auri™ Voice Layer, Fuel Gauge, Injection Visibility Panel, L2 Warm Tier.
—PLANNED — Phase 3: Health Agent, KTM Telemetry and Intelligence System, CIIM™, Dorada Agent, Medical Imaging layer, PAIPy™ interface, Auri Identity™ layer, MQI™ Public Profile.

11.5 Build Roadmap — Priority Order

The priority sequence is determined by one criterion: what produces the most value for Members in the least time, in the right order so that each build item enables the next.

1Real-Time Session Storage™ — Every query and response written to Custos Aurum™ as it happens. This is the chatbot killer — the feature that permanently ends session loss. Everything else builds on persistent memory. Estimated: 40–60 lines of code. Highest return on the entire list.
2Injection Visibility Panel — Shows the Member exactly what memory context was injected into each query. Transparency that builds trust and helps Members understand how their memory is being used.
3Query Classifier — The IRT classification engine that powers Query Preview™. Classifies query type in real time as the Member types.
4Fuel Gauge — Token budget display showing the Member how much context window remains in the current session.
5Project Schema — Database structure for project-scoped memory retrieval. Enables get_project_context() to return richer, more targeted context.
6ChromaDB + Embeddings — Approximately thirty cents one-time to embed the full corpus. Semantic search foundation. Unlocks retrieval by meaning rather than just by keyword.
7Tool-Use Retrieval — MQI™ function calling architecture. Partner AIs call memory functions rather than receiving injected context directly.
8Hybrid Search + Re-ranking — FTS5 and ChromaDB results merged using Reciprocal Rank Fusion. The full retrieval stack.
9Ingestion Question System — Structured prompting during ingestion to improve candidate extraction quality.
